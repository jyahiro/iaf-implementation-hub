# IAF Implementation Hub

<HomeCredibility />

**Plan, govern, and deliver analytics across all seven IAF domains**—with versioned Markdown artifacts, a service catalog aligned to tasks, and checkpoints tied to statute and policy.

The Hub helps public-sector organizations apply the INFORMS Analytics Framework (IAF) using practical, agency-agnostic standards, reusable artifacts, and service pathways.

## What You Can Do Here

- Apply IAF domain standards using implementation-ready guidance.
- Use digital templates (Markdown) instead of static document formats.
- Request support through a structured service catalog.
- Build traceable, governance-ready analytics initiatives.

## Start Here

1. Open [Start Here](./start-here/) for role-based onboarding.
2. Review [Domain Standards](./domains/business-problem-framing).
3. Use the [Implementation Toolkit](./implementation-toolkit/) to complete templates and request services.
4. Review versioned changes in [Release Notes](./release-notes) (current **v0.3.0**).

## Implementation Principles

- **Agency-agnostic:** adaptable across federal, state, and local contexts.
- **Governance-first:** all work products support transparency, auditability, and accountable decision-making.
- **Digitally enabled:** templates are maintained as editable, version-controlled artifacts.
- **Operationally useful:** each domain and service is designed for execution, not just reference.
- **Legally aligned:** implementation is subject to applicable law and policy, including the [Evidence Act](https://www.congress.gov/115/plaws/publ435/PLAW-115publ435.pdf) baseline in [Legal Guardrails](./public-sector/legal-guardrails).

## Note on Source Alignment

Definitions and domain intent are aligned to INFORMS IAF reference materials, while implementation guidance is tailored for public-sector customer execution.
