# Interactive Demo Environment

Use this page to run a dynamic, role-adaptive MVP workflow directly in a browser.

## Before You Start

The demo defaults to **Browser-only mode**, so no backend process is required.

To view this page locally, run:

```bash
npm run start
```

Then open this page and execute:
1. Login and choose role (experience adapts to role)
2. Create project
3. Work domain-by-domain in a guided wizard
4. Enter domain details, save artifact/service actions, and satisfy gate checks
5. Advance through all 7 domains
6. View dashboard and audit events in summary step

The role-specific prompts and task emphasis follow the [Task RACI Matrix](/docs/legal-policy/role-task-matrix/), whose role names and short codes align with the **Microsoft Team Data Science Process (TDSP)** (see [By role](/docs/start-here/by-role/)). Task links in the demo open in a new tab to the referenced task description.

Domain titles in the wizard match the INFORMS IAF domain names in `references/` and the [Task RACI Matrix](/docs/legal-policy/role-task-matrix/).

## Federal Register and eCFR assists (Phase A)

In **Step 2: Project Setup**, the demo can call the public [Federal Register API](https://www.federalregister.gov/reader-aids/developer-resources/rest-api) to suggest recent documents (optional agency filter) and the public [eCFR API](https://www.ecfr.gov/reader-aids/ecfr-developer-resources) to add a **CFR title** browse anchor. Items you confirm are stored as **regulatory anchors** and carry through all seven domains: suggested policy text references them, exported artifacts list them, and you can return to Project Setup to adjust anchors without losing your project.

These calls run in the browser; if your network blocks cross-origin requests, use the demo without anchors or run from an unrestricted connection. Always verify that a suggested document or CFR title applies to your initiative.

## Optional: API-connected mode

If you want to run against the API instead of browser-only mode:

```bash
npm run platform:api
```

Then uncheck **Browser-only mode** in the demo component and set API Base URL if needed.
