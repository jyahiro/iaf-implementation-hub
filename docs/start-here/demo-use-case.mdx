# Demo Use Case: Permit Review Time Reduction

This demo shows how a public-sector team can use the IAF Implementation Hub to reduce permit review cycle time while maintaining governance, transparency, and policy compliance.

<DemoJurisdictionBanner />

## Scenario

A city agency experiences long review timelines for building permit applications. Leadership wants to reduce average review time by 25% without compromising fairness, quality, or legal compliance.

## Desired Outcomes

- Reduce average permit review cycle time by 25%.
- Improve queue visibility for supervisors.
- Standardize escalation for high-risk applications.
- Preserve auditability and explainability for decisions.

## End-to-End IAF Walkthrough

### Domain I: Business Problem (Question) Framing
- Create **Business Problem Brief**.
- Align sponsor, permitting office, legal, and IT stakeholders.
- Complete **[M-13-13 Open Data Policy Governance Checklist](/docs/public-sector/templates/template-governance-domain-01-m-13-13-open-data-policy-checklist/)**.
- Gate decision: approve progression to Domain II.

### Domain II: Analytics Problem Framing
- Translate business need into analytics problem:
  - predict review delay risk
  - identify workflow bottlenecks
- Define success measures (cycle time, rework rate, queue age).
- Record baseline performance.

### Domain III: Data
- Build source inventory (application records, reviewer workload, inspection dependencies).
- Complete data quality checks and lineage notes.
- Complete [OPEN Data/FDS compliance artifact](/docs/public-sector/templates/template-governance-domain-03-open-data-fds-compliance-record/).
- Gate decision: approve progression to Domain IV.

### Domain IV: Methodology (Approach) Framing
- Evaluate options:
  - queue analytics + rules engine
  - risk scoring model + routing recommendations
- Select approach with best explainability and operational fit.

### Domain V: Analytics/Model Development
- Build delay-risk model and workflow recommendation logic.
- Validate performance against baseline and fairness checks.
- Produce model findings and limitations report.
- Complete [AI governance assurance case](/docs/public-sector/templates/template-governance-domain-05-06-ai-rmf-m-24-10-assurance-case/) when applicable.

### Domain VI: Deployment
- Run business validation with permit operations team.
- Pilot in one district before citywide rollout.
- Verify production data flows and escalation controls.
- Gate decision: deployment approval.

### Domain VII: Analytics Solution Lifecycle Management
- Monitor cycle time and error metrics weekly.
- Recalibrate model quarterly or after policy/process changes.
- Review side effects (e.g., workload imbalance by reviewer group).
- Maintain documentation and periodic business-case revalidation.

- Domain VI/VII Operational Lifecycle Review

## Demo Success Criteria

- Measured cycle-time reduction trend within first 60-90 days.
- No critical compliance findings in gate reviews.
- Clear audit trail for approvals, service actions, and artifact changes.
- Operational adoption by permitting supervisors and reviewers.
