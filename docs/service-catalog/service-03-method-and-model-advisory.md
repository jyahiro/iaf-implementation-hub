# Service 03: Method and Model Advisory

## Purpose
Support method selection and model validation decisions across Domains IV and V.

## When to Use
- Teams need an independent method trade-off analysis.
- Model performance is uncertain or difficult to explain.
- Technical approach requires governance review before build or release.

## Required Inputs
- Analytics problem statement
- Candidate methods and constraints
- Model artifacts and evaluation outputs (if available)

## Outputs
- Method selection recommendation
- Model validation and risk summary
- Readiness recommendation for deployment planning
