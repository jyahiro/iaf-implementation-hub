# Release Notes

Versioned changelog for the IAF Implementation Hub (documentation site, templates, and interactive demo). **Current release: v0.3.0.**

[View the latest on GitHub](https://github.com/jyahiro/iaf-implementation-hub/commits/main) for commit-level history.

## Versioning

We follow [Semantic Versioning](https://semver.org/) for the Hub as a product surface:

- **Major (x.0.0):** Breaking changes to URLs, navigation, or templates that require migration.
- **Minor (0.x.0):** New features, new sections, or material demo or platform capability upgrades.
- **Patch (0.0.x):** Fixes, copy edits, link repairs, and small non-breaking improvements.

Until the Hub reaches a stable 1.0 baseline, minor releases may bundle several themes in one version.

## v0.3.0 — 2026-04-28

### Added

- Master role list control in the interactive demo to enforce consistent role references across prompts and workflows.
- Policy-aligned governance gate objectives for Domains I, III, V/VI, and VII in demo domain standards and generated artifacts.
- “Generate Example” actions for open text fields in the demo workflow (context-aware from prior inputs).
- Dummy ICAM directory with type-ahead search and autofill for stakeholder / people fields.

## v0.2.0 — 2026-04-26

### Added

- Interactive demo expansion: role-based task emphasis, form-style task workflows, and policy-linked task references.
- Task-level [RACI matrix](./legal-policy/role-task-matrix) with anchors for deep links from the demo.
- Domain standards alignment controls in the demo and in exported markdown artifacts.
- Legal and policy hyperlinks across the service catalog, domain pages, implementation toolkit, and governance templates.

- Demo task links use canonical doc routes so anchors resolve after navigation (including hosted GitHub Pages).
